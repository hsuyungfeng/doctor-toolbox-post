## 1. System Classification

**System Type:** Content Generation Pipeline

**Description:**
An offline, batch-oriented content generation pipeline that takes Taiwanese Western Medicine (西醫) clinic attributes (clinic name, medical specialty, scraped Facebook Intro, and latest Facebook posts) and uses a local `llama-qwen36` LLM to generate highly personalized, B2B marketing copywriting. The copy promotes "Doctor Toolbox" (an AI SOAP medical records generator) via Facebook Messenger. If a clinic lacks sufficient data (Intro or Posts), the system falls back to pre-approved generic B2B copywriting templates to ensure campaign completeness.

**Critical Failure Modes:**
1. **Specialty Mismatch (Hallucination):** Pitching pediatric features (e.g., child vaccinations) to an orthopedic clinic, destroying B2B trust and conversion rates.
2. **Medical Care Act Violations (Legal Risk):** Generating copy that claims the tool provides clinical diagnosis assistance or guarantees treatment outcomes, violating Taiwan's strict medical advertising laws.
3. **Obvious AI Boilerplate & Length Limit Breaches:** Outputting greeting cliches ("親愛的醫師您好...") or exceeding Messenger character limits (>180 chars), causing users to mark the message as spam.
4. **Data Corruption / Silent Drops:** Failing to parse the LLM's response or dropping clinic records due to encoding/malformed string errors.

## 1b. Domain Context

> Researched by `gsd-domain-researcher`. Grounds the evaluation strategy in domain expert knowledge.

**Industry Vertical:** Healthcare / B2B Medical Marketing (Taiwan)

**User Population:** Outreach operators using automated Messenger sending to contact clinic managers, doctors, and administrative staff.

**Stakes Level:** Medium-High. Output is sent directly to external clinic Facebook accounts. Obvious automation or regulatory violations will lead to direct account reports, resulting in permanent Facebook business page/account bans.

**Output Consequence:** A single illegal medical claim or spam flag can shut down the outreach Facebook profile.

### What Domain Experts Evaluate Against

* **Dimension: Medical Specialty Context Alignment**
  * **Good:** The generated copy links the specific pain points of the clinic's listed specialty to SOAP time savings (e.g., Pediatric clinic gets SOAP child vaccination/development entry context; Ophthalmic clinic gets myopia control records context).
  * **Bad:** The copy refers to general medical records, generic medical services, or confuses specialties (e.g., pediatric vaccine records pitched to an orthopedics clinic).
  * **Stakes:** High.
  * **Source:** Doctor Toolbox clinical supervisor guidelines.

* **Dimension: Taiwan Medical Care Act (醫療法) Compliance**
  * **Good:** The copy only mentions administrative efficiency, transcription, time savings, LINE booking, and billing integration.
  * **Bad:** The copy promises clinical diagnosis accuracy, therapeutic success, or uses prohibited medical superlatives ("最佳", "第一", "根治", "保證").
  * **Stakes:** Critical.
  * **Source:** Article 85 & 86 of the Taiwan Medical Care Act (醫療法第八十五條、八十六條).

* **Dimension: Natural Conversational Flow & Efficacy**
  * **Good:** Traditional Chinese, B2B tone, starts immediately with value ("針對貴診所在...方面的看診特色..."), ends with a clean URL.
  * **Bad:** AI-style templates ("您好，我是您的AI助手...", "希望沒有打擾到您..."), English terms where traditional Chinese is expected, or missing links.
  * **Stakes:** High.
  * **Source:** Sales outreach benchmarks.

### Known Failure Modes in This Domain

* **Superlative Hallucination:** LLM defaults to marketing jargon like "最先進的 AI", "全台第一醫療平台", which triggers immediate administrative fines in Taiwan for unregistered medical advertising.
* **Specialty Hallucination:** Assuming a clinic's department based solely on a vague name (e.g., "王家庭診所" mapped to pediatrics instead of general medicine).

### Regulatory / Compliance Context

* **Taiwan Medical Care Act:** Strictly prohibits non-medical institutions from advertising medical services. B2B software sales must frame themselves as "administrative optimization utilities for medical staff" rather than medical diagnostics.

### Domain Expert Roles for Evaluation

| Role | Responsibility |
|------|---------------|
| Clinical Supervisor (Licensed Doctor) | Calibrates the specialty mapping rubric and verifies clinical terminology correctness |
| Compliance Specialist / Legal Consultant | Reviews the superlative blacklist and approves the generic copy fallback templates |

## 2. Framework Decision

**Selected Framework:** Python `urllib.request` + `pydantic` (Stdlib-centric)

**Version:** Python 3.10+, Pydantic v2.12.x

**Rationale:**
The outreach pipeline is a local, sequential batch script executing under strict rate limits (5-10 minute delays per message). There are no concurrent agent state-machine or multi-agent communication needs, ruling out frameworks like LangGraph or CrewAI. Standard Python `urllib` is already utilized for local LLM endpoint requests. Introducing `pydantic` provides robust structural validation of JSON outputs from the LLM, ensuring strict length, blacklist, and formatting conformance before messages reach the sender loop.

**Alternatives Considered:**

| Framework | Ruled Out Because |
|-----------|------------------|
| OpenAI Python SDK | Added dependency; standard python stdlib `urllib` is already written and runs successfully against the OpenAI-compatible endpoint. |
| CrewAI | Over-engineered for a simple single-turn batch generation pipeline. |
| LangGraph | Over-engineered; there are no branching states or loops in the LLM step. |

**Vendor Lock-In Accepted:** No. The local endpoint uses standard OpenAI completions schema, making it simple to hot-swap to Ollama, vLLM, or commercial APIs by changing the `LLM_API_URL` env var.

### Core Imports

### Entry Point Pattern
```python
from pydantic import BaseModel, Field, field_validator

class ClinicCopySchema(BaseModel):
    personalized_copy: str = Field(description="Outreach copy text in Traditional Chinese, 100-150 chars.")
    specialty_tag: str = Field(description="Identified clinic specialty (e.g., Pediatrics, ENT).")

    @field_validator("personalized_copy")
    @classmethod
    def validate_copy(cls, val: str) -> str:
        # 1. Length check
        if len(val) < 80 or len(val) > 180:
            raise ValueError("Copy length must be between 80 and 180 characters.")
        # 2. Traditional Chinese check
        if any(char in val for char in ["亲", "医", "这", "国"]): # Simple check for simplified chars
            raise ValueError("Copy must be in Traditional Chinese only.")
        # 3. Superlative blacklist check
        blacklist = ["最佳", "最先進", "保證療效", "根治", "全台第一"]
        for term in blacklist:
            if term in val:
                raise ValueError(f"Prohibited regulatory term found: '{term}'")
        return val

# Query pattern
def generate_copy(prompt: str) -> dict:
    url = "http://localhost:8080/v1/chat/completions"
    data = {
        "messages": [{"role": "user", "content": prompt}],
        "temperature": 0.3,
        "response_format": {"type": "json_object"}
    }
    req = urllib.request.Request(
        url,
        data=json.dumps(data).encode("utf-8"),
        headers={"Content-Type": "application/json"}
    )
    with urllib.request.urlopen(req, timeout=60) as response:
        result = json.loads(response.read().decode("utf-8"))
        content = result["choices"][0]["message"]["content"]
        # Validate structured JSON via Pydantic
        parsed = json.loads(content)
        return ClinicCopySchema(**parsed).model_dump()
```

### Key Abstractions

| Concept | What It Is | When You Use It |
|---------|-----------|-----------------|
| `BaseModel` | Core Pydantic schema declaration class | To define structured output fields and types for the LLM response |
| `field_validator` | Custom validation decorator in Pydantic | To enforce regulatory blacklists, length bounds, and character sets |
| `response_format` | API parameter forcing JSON mode | To ensure the local LLM outputs a valid JSON string matching the schema |

### Common Pitfalls
1. **JSON Parsing Failures:** Local models in JSON mode might still prepend preambles ("Here is your JSON:") or fail to escape quotes. *Mitigation:* Extract JSON using regex from raw text if parsing fails directly.
2. **Simplified Chinese Leakage:** Qwen models often slip into Simplified Chinese characters due to pre-training distribution. *Mitigation:* Run character map conversion or validate with Pydantic.
3. **Empty Input Fields:** Scraped fields might be missing. *Mitigation:* Pydantic schemas must declare fallback attributes or allow empty strings gracefully.

## 4. Implementation Guidance

**Model Configuration:**
* **Model:** `llama-qwen36` (configured via Docker locally)
* **Temperature:** 0.2 (lowered from default to maximize formatting compliance)
* **Max Tokens:** 300
* **System Prompt:** Traditional Chinese marketing guidelines, explicitly locking down character limits and blacklist constraints.

**Core Pattern:**
1. Fetch clinic row.
2. If `Intro` and `Latest_Post` are both empty, write `generic_copy` directly (Phase 2 reliability requirement).
3. If data is present, truncate `Intro` and `Latest_Post` to 400 characters to prevent context bloat.
4. Call LLM with JSON mode enabled.
5. Parse and validate output using Pydantic `ClinicCopySchema`.
6. On `ValidationError`, retry up to 2 times with a system warning.
7. If all retries fail, fall back to the pre-approved `generic_copy` to guarantee row complete.

**Tool Use:**
No external tool calling is allowed by the LLM inside `generate_copy_llm.py` to keep generation speeds high and deterministic.

**State Management:**
Progress state is persisted incrementally in `clinics西醫.csv` every 10 clinics. Graceful SIGINT handlers ensure that if the operator hits `Ctrl+C`, the database is flushed to disk before exiting.

**Context Window Strategy:**
Since inputs are limited to a single clinic's scraped text, the window size per call is <1,500 tokens. No advanced summarization or sliding window is needed.

## 4b. AI Systems Best Practices

> Written by `gsd-ai-researcher`. Cross-cutting patterns every developer building AI systems needs — independent of framework choice.

### Structured Outputs with Pydantic

Integrating structured outputs prevents downstream automation scripts from breaking due to missing fields. The following pattern must be applied inside `generate_copy_llm.py`:

```python
def generate_with_retry(clinic_name, dept, intro, post, retries=2):
    prompt = build_prompt(clinic_name, dept, intro, post)
    for attempt in range(retries + 1):
        try:
            raw_response = call_local_llm(prompt)
            if not raw_response:
                raise ValueError("Empty LLM response")
            
            # Extract JSON block if LLM returned markdown wrapping
            if "```json" in raw_response:
                raw_response = raw_response.split("```json")[1].split("```")[0].strip()
            
            data = json.loads(raw_response)
            validated = ClinicCopySchema(**data)
            return validated.personalized_copy
        except (json.JSONDecodeError, ValidationError, ValueError) as e:
            print(f"  ⚠️ Validation failed on attempt {attempt+1}: {e}")
            if attempt == retries:
                print("  ❌ All retries exhausted. Using generic fallback copy.")
                return get_generic_fallback_copy(dept)
```

### Async-First Design
Because the downstream Facebook outreach script (`send_outreach.py`) is throttled to a **5-10 minute random delay** to mimic human typing and avoid account bans, async concurrency is intentionally **avoided** in both generation and sending. Sequential processing ensures safety.

### Prompt Engineering Discipline
* **Formatting Separation:** System instructions and rules are clearly marked with Markdown headers.
* **Few-Shot Prompting:** The prompt includes exactly two Traditional Chinese few-shot examples (one Pediatrics, one general practitioner) showing correct tone, length, and call-to-action formatting.

## 5. Evaluation Strategy

### Dimensions

| Dimension | Rubric (Pass/Fail) | Measurement Approach | Priority |
|-----------|--------------------|---------------------|----------|
| **Specialty Alignment** | PASS: Pitch matches the clinic's actual department.<br>FAIL: Mentioning unrelated treatments or confusing specialties. | LLM Judge (Promptfoo) | Critical |
| **Regulatory Compliance**| PASS: No superlative claims or therapeutic guarantees.<br>FAIL: Contains blacklist keywords or clinical efficacy claims. | Code (Regex Blacklist) | Critical |
| **Formatting Conformance**| PASS: Copy is between 100-150 characters, Traditional Chinese, and lacks AI boilerplate.<br>FAIL: Missing URLs, simplified chars, or generic greetings. | Code (Pydantic / Regex) | High |
| **Halt Rate Limits** | PASS: Script correctly backs off or exits on block signals.<br>FAIL: Sends messages repeatedly during active block. | Integration Test | High |

### Reference Dataset

**Composition:**
* **10 Specialty Clinics:** Mixed specialties (Pediatrics, ENT, Ophthalmology, Dentistry, Orthopedics) with rich scraped info.
* **5 Sparse Clinics:** Empty intro/post fields to verify generic fallback routing.
* **5 Edge Case Clinics:** Non-standard specialty names or multi-specialty clinics.
* **5 Adversarial Cases:** Inputs containing Simplified Chinese, empty strings, or names containing blacklisted words.

**Labeling:**
Ground truth labels for specialty matching are calibrated and signed off by the Doctor Toolbox founder.

## 6. Guardrails

### Online (Real-Time)

| Guardrail | Trigger | Intervention |
|-----------|---------|--------------|
| **Superlative Blacklist** | Regex hit on "最", "治癒", "第一" | Discard LLM output; fallback to pre-approved generic copy. |
| **Length Checker** | String length > 180 characters | Discard LLM output; fallback to generic copy. |
| **Facebook Ban Signal** | Detect "無法傳送", "暫時限制" on page | Immediately raise SIGINT/kill the process (circuit breaker). |

### Offline (Flywheel)

| Metric | Sampling Strategy | Action on Degradation |
|--------|------------------|----------------------|
| **Specialty Accuracy** | Manual 10% review of sent copy | Refine system prompt few-shot examples if error rate > 2%. |
| **Reply Rate** | Monthly audit of `outreach_sent_log.jsonl` | Adjust prompt hook if clinic reply rate drops below 1.5%. |

## 7. Production Monitoring

**Tracing Tool:** Local JSONL audit log (`outreach_sent_log.jsonl`).

**Key Metrics to Track:**
* LLM API failure/timeout rate.
* Pydantic validation failure count.
* Ratio of personalized copy vs. generic copy fallback.
* Daily total message volume and block flags.

**Alert Thresholds:**
* Halt outreach immediately if Facebook block events exceed 2 within a single 24-hour campaign.
* Terminate generation if LLM timeout rate exceeds 20% in a 10-clinic sliding window.
